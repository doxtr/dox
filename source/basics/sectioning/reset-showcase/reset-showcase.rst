This is how a part is now rendered when included from a 'section' level section.

This section uses the 'part' sectioning level marker. As it is however included from a toctree in a section that is using the 'section' sectioning level marker, it will be rendered as a 'sub-section' level section. This allows you to reset the sectioning levels for a specific part of the documentation, while still maintaining the overall structure of the documentation.

This is useful when you want to include a part of the documentation that has a different structure than the rest of the documentation, or when you want to include a part of the documentation that is not related to the rest of the documentation. 

For example, you can use this to include a part of the documentation that is focused on a specific topic, or to include a part of the documentation that is focused on a specific audience.

This lets you import a document that was structured the same way from another author as a 'chapter', 'section' or 'sub-section' without having to change the sectioning levels in the imported document, and without affecting the sectioning levels in the rest of the documentation. This allows for better organization and structure for the documentation, and also allows for better navigation for the readers.
